# Product and Admissions Roadmap

## Release 0.1 — Repository foundation — included

- Docker development environment
- React application
- Django API
- PostgreSQL and Redis
- accounts, roles and governance identities
- collection request workflow
- public impact page
- tests and CI

## Release 0.2 — Controlled household pilot

- email verification
- improved draft editing
- image uploads through object storage
- administrator assignment screens
- pilot with 10–20 households
- structured feedback and defect log

## Release 0.3 — Volunteer operations

- volunteer acceptance and completion views
- calendar and reminder workflows
- safety acknowledgement
- workload and reliability reporting

## Release 0.4 — Recycler traceability

- receipt upload hardening
- verified-weight reconciliation
- automatic impact updates from completed handovers
- recycler-facing confirmation option

## Release 0.5 — Corporate campaigns

- organization enrollment
- CSV inventory import
- campaign coordinator dashboard
- downloadable impact certificate

## Release 0.6 — Optimization study

- geocoding provider
- map interface
- request clustering
- vehicle capacity and time windows
- baseline-versus-optimized route evaluation

## Release 0.7 — Production hardening

- MFA/passkeys
- rate limiting
- object storage and file scanning
- centralized logs and error monitoring
- encrypted off-site backups
- accessibility audit
- privacy review

## Release 1.0 — Public capstone release

- stable real-user deployment
- evaluation report
- public sanitized repository
- architecture and security documentation
- demo video
- open-source replication guide

## Advanced feature selection

Build only one advanced research feature after the workflow is stable:

1. pickup-demand forecasting; or
2. image-assisted device categorization.

Each requires a baseline, a dataset, evaluation metrics, limitations and documented human oversight.
