# T10 Capstone Evidence Plan

## Core claim

EcoRevive OS converts an existing community-led e-waste initiative into a traceable digital operating platform for residents, companies, volunteers, administrators and certified recycler handovers.

## Evidence to collect

- screenshots and release tags showing product growth;
- architecture decision records;
- meaningful issues and pull requests;
- automated test results;
- anonymized pilot usage;
- baseline-versus-system operational comparisons;
- recycler-verified impact evidence;
- user feedback and design changes;
- documented failures and technical tradeoffs;
- a public demo using synthetic data;
- a final capstone report and 4–6 minute demonstration video.

## Interview questions to master

- Why Django and a modular monolith?
- How is authorization enforced?
- How does the state machine prevent invalid workflows?
- How are exact addresses protected?
- How does route optimization work and what is its baseline?
- What did AI generate, and what did you change?
- Which design failed and why?
- How did real user behavior change the product?

## Warning

A polished interface without real use, evaluation and ownership evidence is weaker than a smaller system with verified deployment, testing and honest reflection.
