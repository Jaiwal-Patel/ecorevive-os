# Admissions Evidence Matrix

| Application claim | Strong evidence to preserve |
|---|---|
| Identified a real community problem | original manual workflow, anonymized requests, stakeholder interviews |
| Built the platform | commit history, issues, pull requests, architecture decisions, tests |
| Used AI responsibly | AI assistance log, code review changes, tests, explanations |
| Software had real users | consented pilot metrics, anonymized screenshots, operator verification |
| Improved operations | baseline and post-deployment timing, completion and route measures |
| Produced environmental impact | recycler receipts, reconciled weights, methodology |
| Demonstrated leadership | volunteer onboarding, admin handover, corporate/community coordination |
| Demonstrated CS depth | authorization design, data model, state machine, optimization evaluation |
| Learned from failure | defect log, retrospective, redesign decisions, limitations |
| Made work reusable | documentation, Docker setup, demo data, replication guide |

## Evidence discipline

Keep raw private evidence outside the public repository. Store public artifacts only after anonymization and maintain a mapping in the private evidence register.
