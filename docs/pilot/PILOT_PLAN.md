# Controlled Pilot Plan

## Objective

Determine whether EcoRevive OS can coordinate real collections more reliably and efficiently than the existing combination of calls, WhatsApp messages and spreadsheets.

## Participants

- 10–20 consenting households;
- 2–4 volunteers;
- one operations administrator;
- one corporate collection if feasible;
- recycler evidence for completed handovers.

## Measures

- request completion rate;
- time from submission to approval;
- time from approval to assignment;
- missed or rescheduled pickups;
- distance per completed collection;
- completeness of recycler traceability;
- user task-completion rate;
- participant satisfaction and qualitative feedback.

## Safeguards

Use real data only in the private production environment. Use generated seed data in screenshots, demonstrations and the public repository.

## Exit criteria

Do not expand the pilot until critical security defects are closed, backup restore has been tested and the operations team can complete the full workflow without founder intervention.
