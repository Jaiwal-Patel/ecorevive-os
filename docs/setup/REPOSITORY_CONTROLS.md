# Recommended GitHub Repository Controls

After the first successful push:

1. Keep the repository private during the real-user pilot.
2. Enable Dependabot alerts and secret scanning when available.
3. Protect `main` and require a pull request before merge.
4. Require the `CI / backend` and `CI / frontend` checks.
5. Block force pushes and branch deletion.
6. Use GitHub Issues for every material feature or defect.
7. Never upload real participant exports, photographs or database backups.
8. Create signed release tags for pilot milestones.
9. Sanitize sample data before a public release.

A GitHub role is not a substitute for documented organizational ownership of the domain, cloud account, backups and email/WhatsApp assets.
